## For Claudes to come

**CC's plugin model is whole-plugins-only.** There is no per-skill toggle — not in settings.json, not in plugin.json schema, not via permissions. The `enabledPlugins` field controls whole plugins; skills inside a plugin live or die with it. This is the load-bearing constraint for any skill-suite design. Implications:

1. If you want fine-grained context-scoped enablement (e.g. SPSS skill only when working on SPSS data), the only mechanism available is splitting your plugin into multiple smaller plugins. Trousse-as-kitchen-roll trades scoping precision for metaphor coherence.
2. The only way to remove individual skills from a suite is to edit the source. There's no "user-disabled" state for sub-plugin skills.
3. Built-in CC skills (`update-config`, `keybindings-help`, `simplify`, `fewer-permission-prompts`, `loop`, `claude-api`) are baked into the binary and cannot be disabled at all without `--bare` (which also strips CLAUDE.md, hooks, plugin sync, auto-memory — too nuclear for daily use).

**Description length is register, not just tokens.** Skills with 1500-char descriptions stacked 30-deep at session-start create a "demands accumulating" register that biases toward compliance/anxiety rather than relaxed engagement. Trim aggressively. Targets: action verb opener, timing gate (BEFORE/FIRST/MANDATORY/WHEN), trigger phrases in quotes, `(user)` tag if user-invocable. Drop: implementation detail, mode breakdowns, feature flexes ("cross-platform", "no containers"), full lists where examples suffice. Aim for ~400 chars. CSO score often *improves* from the trim because the trigger signal stops competing with noise (ardoise went 95 → 100).

**Verify the mechanism before designing the scope.** When proposing per-X scoping (per-skill, per-project, per-anything), confirm the underlying system actually supports it before drafting recommendations. The cost of an unverified proposal is mid-execution walk-back, which burns user trust more than a slower upfront proposal would. Self.md already names this; the pull is real and back-pressure helps.
